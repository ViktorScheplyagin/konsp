----Комментарии и синтаксис
Атрибут title у элемента - при наведении появляется "подсказка" со значением атрибута.

----HTML оформление текста
<s></s> - делает текст зачеркнутым
<u></u> - делает текст подчеркнутым
<b> просто делает текст жирным. 
<strong> используется для выделения важной информации. Например, некоторые браузеры для слепых прочитают такой текст более низким тоном.
Тег <em> используется для создания акцента на подчеркнутой фразе.
Тег <i> просто подчеркивает слова. Например термины, названия и т.д.

----HTML списки
Упорядоченный список. По умолчанию нумеруется арабскими цифрами.
	<ol>
		<li>...</li>
		...
	</ol>
	Атрибуты (тех же эффектов можно добиться в css):
		type - тип нумерации. Значения:
			"A" - нумерация по алфавиту (заглавные)
			"a" - маленькие буквы.
			"I" - римскими цифрами.
			"i" - римские цифры обозначаются маленькими буквами.
		start - указать значение, с которого начнется нумерация списка
		reversed - нумерация с конца.
Маркерованный список
	<ul>
		<li>...</li>
	</ul>
	Атрибуты:
		type - тип маркера. Значения:
			"disc" - черный заполненный кружок. Задан по умолчанию.
			"circle" - пустая окружность.
			"square" - заполненный квадратик
Список определений - в нем сначала отображается термин, потом, с новой строки, с абзаца, определение
	<dl>
		<dt>Термин</dt>
		<dd>Определение</dd>
	</dl>

----Картинки и иконки сайта

----Ссылки на сайте
Типы ссылок
	Внешняя - ведет на другой домен
	Внутренняя - на другую страницу того же домена
	Якорь - на участок той же страницы
Атрибуты тега <a>
	target - как и где откроется ссылка. Значения
		"_blank" - ссылка откроется в новом окне браузера
		"_self" - в текущем окне
		"_parent" - в родительском фрейме
		"_top" - отменяет все фреймы и открывается в полном окне браузера.
	rel="nofollow" - поисковая система не будет индексировать данную ссылку.
Создание якоря
	<a id="there"></a> - ставим ссылку на том месте, куда надо будет перейти
	<a href="#there">Сюда</a> - эта ссылка будет вести на то место на странице, где расположена вышеуказанная.
<base> - одиночный тег. Указывается в <head>. Имеет те же атрибуты, что и ссылки. Применяет их ко всем ссылкам на данной странице.

----Основы таблиц
<table>
	<caption>Cap!</caption> - заголовок над таблицей.
	<col> - управление колонками таблицы
	<thead> - Указывает семантически на шапку таблицы. Может быть только 1 на всю таблицу. Всегда будет отображаться сверху таблицы.
		<tr>...</tr>
	</thead>
	<tfoot> - Указывает семантически на подвал таблицы. Может быть только 1 на всю таблицу. Всегда будет отображаться снизу таблицы.
		<tr>...</tr>
	</tfoot>
	<tbody> - Указывает семантически на тело таблицы. Может быть сколько угодно в таблице.
		<tr>
			<th>CapCell</th> - заглавная ячейка. Текст в ней жирный, выравнивается по центру.
			...
		</tr>
	</tbody>
</table>
Атрибуты <td>
	rowspan - в скольких соседних строках объединить ячейку с данной. 
	colspan - в скольких соседних столбцах объединить ячейку с данной.
	В обоих случаях, на самом деле, ширина/высота данной ячейки становится равной сумме соседних объединяемых.
Тег <col> - если прописать несколько подряд, то каждый будет управлять соответствующей колонкой (которая такая же по счету). Атрибуты
	span - значение означает, на сколько соседних колонок будет распростраться управление.
<colgroup> - разрешается писать как одиночный тег, если используется для применения к, непосредственно, колонкам таблицы. Если служит контейнером для <col>, то нужно его закрывать. Отличается от <col> только тем, что при назначении атрибута rules="groups" тегу <table>, браузер будет рисовать границу только между колонками, к которым относится, непосредственно, <colgroup>

----Формы
Атрибут тега <form>
	name - должен иметь уникальное значение. Служит для доступа к форме из JS
	accept-charset - указываем кодировку, в которой данные будут отправлены на сервер. По умолчанию используется та, которая указана в шапке страницы
	enctype - указывает вид кодирования данных, который будет использоваться при отправке их на сервер. Значения
		"application/x-www-form-urlencoded" - русские буквы кодируются в 16-ричном виде, вместо пробелов ставится "+". Латиница не кодируется.
		"multipart/form-data" - данные вообще не кодируются.
		"text/plain" - кодируются только пробелы (+)
	method - метод, которым данные будут отправляться на сервер
		"GET" - данные до 4кБ
		"POST" - большие объемы данных
	novalidate - без значения. Запрещает валидацию формы
	autocomplete - подсказка заполнения формы.
		"on" - включено
		"off" - выключено

----input
Атрибуты
	value - значение по умолчанию. Для submit - это название кнопки. Для текстового (text) поля - значение внутри него.
	placeholder - подсказка для текстового поля.
	required - без значения. Поле должно быть заполненным.
	type="email" - проверяет, введен ли электронный адрес.
	type="reset" - кнопка очищает всю форму.
	autofocus - поле получает фокус автоматически, после загрузки страницы.
<label for="значение id input'а" - текст, относящийся к конкретному полю. При клике, поле с заданным в for, id'ом, получает фокус.
	Так же можно инпут вписать внутрь лейбла с текстом.
Чтобы выбирался только один из переключателей, нужно у всех их задать атрибут name с одинаковым значением.

----url, телефон, пароль, поиск
type="tel" - поле для ввода телефона
maxlength - максимальное количество символов, которое може быть введено в поле
type="url" - поле для ввода урла. Браузер проверяет введен ли урл.
type="search" - поле поиска. Работает только при наличии обработчика.
type="password" - поле для ввода пароля. Замещает введенный символы точками.
type="number" - поле, поддерживающее ввод только чисел
	min - атрибут, указывающий минимальное значение, которое может быть введено.
	max - максимальное
	step - числовой шаг, с которым может уменьшаться/увеличиваться значение в поле.
type="range" - "ползунок". Применяются те же атрибуты, что и для type="number"
<output></output> - область, в которую выводится информация, в основном, с помощью скриптов. Атрибуты:
	for - id одного или более элементов, для связывания с output'ом
	form - имя формы, которой принадлежит область вывода.
	name - уникальное имя.

----Поля ввода даты и времени
type="date" - поле для ввода даты
type="time" - поле для ввода времени
type="month" - месяц
type="week" - неделя
type="color" - поле для выбора цвета
Атрибут pattern - с помощью regexp, устанавливает маску для вводимой в поле информации.

----select, option, optgroup
size - атрибут у select, указывает количество отображаемых пунктов. Если пунктов больше - остальные доступны по скроллу.
multiple - для select означает что можно выбрать сразу несколько пунктов.
<optgroup></optgroup> - тег для группировки <option>. Пункты в нем будут отображаться в списке с отступом слева и отступом сверху, в виде пустой строки. Эту пустую строку можно подписать, с помощью атрибута label.
	Атрибут disabled делает все пункты, внутри него, недоступными

<textarea> - поле для ввода большого объема текста. Атрибуты
	cols - ширина в символах
	rows - высота в строках
	required - обязательно для заполнения
	disabled - недоступно для заполения
	placeholder - подсказка
	Прочие атрибуты для полей ввода.

----fieldset, legend
<fieldset> - обрамляет содержащиеся поля в рамку. Атрибуты
	disabled - все поля внутри заблоированы.
	title - подсказка при наведении
<legend> - пишется внутри fieldset, в самом верху. Содержимоие появляется на созданной филдсетом рамке.
